预测随机数

到此，我们便可以通过此方法来预测随机数，但是我们发现我们上述所述都是指随机种子固定且为0的情况，那要是随机种子不固定呢? 我们又该如何处理:

设初始随机数列表为 s[31], 生成的随机数列表o[n]在有限域 0x80000000 下 （0xffffffff + 1）(INT_MAX)(RAND_MAX) 有

也就是说在 31 个随机数生成之后，原来的随机数列表s会焕然一新，此时我用s'来代表新的初始随机数表。

此时会有：o[31] = o[0] + o[28]或o[31] = o[0] + o[28] + 1两种情况，为什么会有如此情况呢

也就是说 当一个数为奇数时，它的二进制位末位为 1， 此时我们右移一位时，会将末尾的 1 移去；当一个数为偶数时，它的二进制位末位为 0， 此时我们右移一位时，会将末尾的 1 移去。

在此基础上会有如下四种情况：

  1. 当 s[28] + s[0]的二进制位末位为 0， s[0] + s[3]的二进制位末位为 0， 对应的o[31]右移之前的二进制位末位为 0：

    此时 o[31] = o[0] + o[28]

  2. 当 s[28] + s[0]的二进制位末位为 0， s[0] + s[3]的二进制位末位为 1， 对应的o[31]右移之前的二进制位末位为 1：

    此时 o[31] = o[0] + o[28]

  3. 当 s[28] + s[0]的二进制位末位为 1， s[0] + s[3]的二进制位末位为 0， 对应的o[31]右移之前的二进制位末位为 1：

    此时 o[31] = o[0] + o[28]

  4. 当 s[28] + s[0]的二进制位末位为 1， s[0] + s[3]的二进制位末位为 1， 对应的o[31]右移之前的二进制位末位为 0：

    此时 o[31] = o[0] + o[28] + 1

也就是说，当我们有了o[0]即o[28]的值之后，我们有大于50%的几率预测o[31]，进一步，我们可以发现 o[n] = o[n-31] + o[n-3]或o[n] = o[n-31] + o[n-3] + 1
